# Formatting-Style

## 개요

코드의 시각적 형태를 통일하여 가독성을 높이고, 불필요한 diff 를 최소화하기 위한 포맷팅 규칙입니다.

## 기본-설정

### 인덴트
- **Tab** 문자를 사용합니다. (Space 아님)
- Tab 크기는 **4칸**으로 설정합니다.

### 줄-길이 (Column limit)
- 한 줄은 **최대 100자**를 권장합니다.
- 주석은 80자 이내로 작성하면 가독성이 좋습니다.

## 공백과-줄바꿈

### 수직-공백 (빈 줄)
- 코드를 논리적 단위(블록)로 구분하기 위해 빈 줄을 적절히 사용합니다.
- 단, 블록이 시작된 직후에는 빈 줄을 넣지 않습니다.

```lua
-- Good:
local Foo = require("./Common/Foo")

local function gargle()
    -- gargle gargle
end

Foo.frobulate()
Foo.frobulate()

Foo.munge()
```

### 수평-정렬
- `=` 위치를 맞추기 위한 수평 정렬은 지양합니다. 유지보수 시 변경 범위가 커지고 diff 가 지저분해집니다.

```lua
-- Good:
local frobulator = 132
local grog = 17

-- Bad:
local frobulator = 132
local grog       =  17
```

### 수식-운영
- 연산자 양옆에는 가독성을 위해 공백을 둡니다. 우선순위를 명확히 해야 할 때만 붙여 씁니다.
- 콤마(`,`) 뒤에는 항상 공백을 둡니다.
- 괄호 `()`, 중괄호 `{}` 등 블록을 여는 문법 요소는 **같은 줄**에 작성합니다.

```lua
print(5 + 5 * 6^2)
```

### 테이블-공백
- 테이블 정의 시 여는 중괄호 `{`를 **새로운 줄**에 두지 않습니다.
- 항목마다 **후행 콤마(trailing comma)** 를 남겨두어, 추후 diff 를 깔끔하게 유지합니다.
- 너무 긴 키-값 구조는 서브 테이블로 분리하는 편이 좋습니다.
- 단순 리터럴 테이블은 한 줄 작성이 가능하나, 변수와 섞여 있다면 여러 줄로 나눕니다.

## 구문-스타일

### 따옴표
- 문자열은 **큰따옴표(`"`)** 사용을 원칙으로 합니다.
- 작은따옴표(`'`)는 가독성 혼동 우려가 있어 최소화합니다.

### 세미콜론
- **세미콜론(`;`)** 은 사용하지 않습니다.

### 한-줄-구문-지양
- `if state then return end` 처럼 한 줄에 몰아 쓰는 것을 피합니다.
- 명확하게 줄을 나눠 가독성을 높입니다.

```lua
-- Good:
if state then 
    return 
end
```
