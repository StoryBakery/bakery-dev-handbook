클라이언트의 즉각적인 반응성과 서버 권위를 동시에 유지하는 것을 목표로 합니다.

## 목표

- 플레이어 입력 직후 체감 반응성을 유지합니다.
- 최종 게임 상태는 서버가 결정하고 모든 클라이언트에 동일하게 복제합니다.
- 예측과 실제 결과가 다를 때 빠르게 정정하여 시각적 충돌을 줄입니다.

## 기본 흐름

1. 클라이언트가 입력 즉시 로컬 예측을 적용합니다.
1. 클라이언트가 서버로 요청을 전송합니다.
1. 서버가 요청을 검증하고 상태를 확정합니다.
1. 서버가 확정 상태를 관련 클라이언트에 복제합니다.
1. 클라이언트가 예측값과 서버 확정값 차이를 정정합니다.

## 복제 규칙

- 판정, 보상, 진행 상태는 서버에서만 계산하고 복제합니다.
- 클라이언트 전용 연출(UI 애니메이션, 카메라, 이펙트)은 로컬에서 처리합니다.
- 복제 데이터는 필요한 범위만 보내고, 수신 측에서 재구성 가능한 값은 줄입니다.

## 정정 규칙

- 서버 확정값이 도착하면 로컬 예측보다 서버 값을 우선합니다.
- 위치/상태 보정은 한 프레임 순간이동보다 짧은 보간 정정을 우선합니다.
- 반복적으로 정정이 발생하는 입력은 예측 범위를 줄이거나 검증 조건을 강화합니다.

## `workspace:GetServerTimeNow()`

서버 권한 환경에서 재시뮬레이션 중에도 `time()` 처럼 과거 시간으로 되돌아가지 않습니다.

`simulationAccess: true` 가 아닌 속성이기 때문입니다

장점:

- `time()`보다 소수점 이하 값이 정확하고 연속적으로 변합니다.
- 같은 프레임 안에서도 더 정밀한 시간 변화를 확인할 수 있습니다.

커스텀 롤백 이나 시뮬레이션 시스템을 만들 때 굉장히 유용합니다.
