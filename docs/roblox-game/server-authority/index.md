로블록스의 서버 권한 검증 에 대해 다루는 문서입니다.
만약 고정틱 시뮬레이션만 사용하는 경우 참고용으로만 사용해주세요.

로블록스에서 서버 권한은 정확하게는 단순히 '서버 권한'이라기 보다는, **롤백 넷코드** 에 가깝습니다.

## 활성화하기

활성화하는 법은 `workspace.AuthorityMode` 를 `Server` 로 설정하는 것입니다.

## 클라이언트에서 미리 예측하기

서버 권한의 핵심 원리는 클라이언트가 특정 인스턴스들에 대해 서버보다 중요 인스턴스들을 몇 프레임 앞서 예측한다는 것입니다.

그리고 미리 예측하다 서버로부터 받은 값이 틀린 속성들이 있다면,
틀린 프레임부터 현재 프레임까지 다시 `BindToSimulation` 에 연결한 콜백을 모두 재실행하는 롤백을 합니다.

또한 **서버는 일방향이며 클라이언트만 오로지 롤백**합니다.

물론 커스텀화된 롤백 시스템에선 다르게 동작하게 만들 수 있지만,
기본 로블록스의 롤백 시스템은 서버의 성능을 가볍게하고 서버가 진실의 원천이며,
클라이언트에서 일부 인스턴스들에 대해 재시뮬레이션 해서 재시뮬레이션하는 방식입니다.

서버에서의 지연보상 개념은 커스텀화된 롤백 시스템을 만들어야합니다.

## 용어

`Resimulation` 과 `Rollback` 은 비슷하나 뉘앙스는 다음과 같습니다:

- `Resimulation`: 시뮬레이션의 환경/상태
- `Rollback`: 이벤트
