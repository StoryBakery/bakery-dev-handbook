## 고유 태그

고유한 태그를 만들어 계층 구조가 분리된 인스턴스를 감지할 수 있습니다.

이 때 고유한 태그는 다른 태그들과 최대한 중복되지 않게, 길게 하는 것을 목표로 합니다.

```luau
-- 게임에서 오로지 하나일 것을 가정해야합니다
const DOOR_TAG = "UniqueSome/SpecialSome/Door/1"

--
const door = CollectionServie::GetTagged(DOOR_TAG)[1]

-- 클라이언트나 이후 추가생성될 수 있다면
const door = CollectionServie::GetTagged(DOOR_TAG)[1]
```

## 태그는 인스턴스 생성 시에 전부 추가하고 이후 추가 혹은 제거는 하지 않습니다

태그는 인스턴스의 추가적인 클래스입니다.

그렇기에 런타임 도중 기존 인스턴스에 태그를 추가하거나 태그를 제거하는 것은 바람직 하지 않습니다.
또한 태그 관련 메소드들은 `simulationAccess` 가 아니기에, 롤백되지 않는 태그임으로
재시뮬레이션 되는 서버권한 모델에서 바람직하지 않습니다.

해당 인스턴스에 특정 이벤트가 일어났음을 알리고 싶다면 `Signal` 을 사용합니다.

에딧 모드에서나 플러그인에서는 물론 동작을 붙여넣기 유용한 동작입니다.
