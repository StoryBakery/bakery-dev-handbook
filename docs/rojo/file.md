## rojo 에서는 rbxm 을 아래에 두면 인스턴스로 변환됩니다

다음처럼 코드에 인스턴스를 `.model.json` 으로 표기하기 힘든 경우 `rbxm` 파일을 넣어 표현할 수도 있습니다

```sh
src/
  Game/
    GameCharacter.rbxm
```

이러면 소스코드로 수정이 어렵지만, 스튜디오에서 수정하고
`rojo syncback` 등을 이용해서 다시 저장할 수 있습니다.

## 플레이스는 rbxl, 모델은 rbxm 으로 저장합니다

`.rbxl` 은 플레이스 파일, `.rbxm` 은 모델 파일입니다.

## xml 포맷은 사용하지 않습니다

xml 포맷인 `.rbxm` 와 `.rbxmx` 은 읽을 수는 있으나, 용량 문제가 있으며 읽기 쉽지도 않습니다.
디버그를 제외하고는 사용하지 않습니다.
