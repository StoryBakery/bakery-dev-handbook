
## 개요

문서를 어떻게 작성하는 지에 대한 가이드입니다.

## 문법

문장 마지막에 점은 무조건 찍습니다.

## 헤더

모든 문서는 `## 개요`로 시작합니다.

`# 제목` 은 사용하지 않습니다.
제목의 경우엔 
```md
---
title: 제목
---

## 개요
```
로 작성합니다.

`#` 은 빌드 파일에 자동생성되기 때문.
raw 마크다운 파일로 렌더링된다할지라도 일관성을 위해 수동으로 작성 시 `#` 은 사용하지 않습니다.

## 텍스트-스타일링-금지 
볼드 처리를 할 때 헤더 제목 전체를 볼드 처리를 하지 않습니다.

```md
<!-- Bad -->
## **볼드-헤더**

<!-- Good -->
## 그냥-헤더
```

## 공백은-대시로

단어와 단어 사이의 공백은 대시(`-`)로 표기합니다.

이유는 마크다운 문서가 HTML 로 변환될 때 공백이 `%20` 으로 바뀌어 URL 이 길어지는 현상을 방지하기 위함입니다.
링크가 어려워지기 때문.

```md
<!-- Bad -->
## 문서 작성 가이드
웹사이트에서 이 헤더 앵커 링크는 https://도메인/문서%20작성%20가이드.md 이 됩니다

<!-- Good -->
## 문서-작성-가이드
웹사이트에서 이 헤더 앵커 링크는 https://도메인/문서-작성-가이드.md 이 됩니다
```


## 링크

문서 내 링크는 반드시 현재 문서를 기준으로 한 상대 경로(`../path/to/doc.md`)로 작성합니다.
저장소 루트 절대 경로나 외부 URL을 사용하는 방식은 허용하지 않습니다.
