## 기본 철학

최소한의 실행 가능한 문서: 낡고 방대한 문서보다는, 항상 최신 상태로 유지되는 작고 정확한 문서가 훨씬 가치 있습니다. 불필요한 내용은 자주, 조금씩 삭제하세요.

완벽함보다 꾸준한 개선: 코드 리뷰와 마찬가지로 문서도 빠른 반복 작업이 중요합니다. 사소한 논쟁에 시간을 낭비하지 말고, 작은 개선을 지속적으로 반영하세요.
