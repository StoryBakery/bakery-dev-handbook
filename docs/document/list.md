title: 목록 작성 스타일
---

## 기본 규칙

- 불릿 목록 마커는 `-`를 사용합니다.
- 목록 마커 뒤에는 공백 1칸을 둡니다.
- 최상위 목록은 들여쓰기 없이 시작합니다.

## 들여쓰기 패턴

- OL(`1.`) 하위 항목은 보통 `+3칸` 들여쓰기.
- 더 깊은 중첩에서도 같은 증가폭을 반복합니다.
  - UL 체인: `0 -> 2 -> 4`
  - OL 체인: `0 -> 3 -> 6`
- 줄바꿈 본문은 목록 본문 시작 위치에 맞춰 정렬합니다.

## UL (Unordered List)

`-` 로 시작합니다.

UL(`-`) 하위 목록은 보통 `+2칸` 들여씁니다

```md
- 상위 불릿
  - 하위 불릿
    - 3단계 불릿
```

## OL (Ordered List)

`1.` 로 시작합니다.

OL(`1.`) 하위 항목은 보통 `+3칸` 들여씁니다

```md
1. 상위 순서 목록
   1. 하위 순서 목록
   1. 하위 순서 목록
```

### 번호 목록

`1.`, `2.`, `3.` 순차 번호와 `1.` 반복 번호를 모두 사용할 수 있습니다.

### 긴 목록에는 Lazy numbering 사용

변경 가능성이 큰 긴 절차는 `1.` 반복 번호를 권장합니다.

```md
1. 상위 순서 목록
   1. 하위 순서 목록
   1. 하위 순서 목록
1. 다음 상위 항목
```

Markdown은 결과 HTML이 번호가 매겨진 목록을 올바르게 렌더링할 수 있을 만큼 똑똑합니다.
내용을 변경할 수 있는 긴 목록, 특히 내용이 긴 중첩 목록의 경우에는 "동일한(lazy)" 번호 매기기를 사용하세요:

```md
1. Foo
1. Bar
  1. Foofoo
  1. Barbar
1. Baz
```

하지만 목록이 짧고 변경될 것으로 예상하지 않는 경우에는 소스에서 읽기 더 좋기 때문에 숫자가 순서대로 매겨진 목록을 사용하는 것을 선호하세요:

```md
1. Foo
2. Bar
3. Baz
```

## 트리

`|` 와 `|-` 의 조합으로 작성합니다.

```md
Model1
|- Object1_1
|- Object1_2
|- AnotherObject1
|- AnotherObject2
|  |- AnotherDescendant1
|  |- AnotherDescendant2

Model2
|- Object2_1
|- Object2_2
```
