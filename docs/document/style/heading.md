## H1 헤딩 사용금지

`# 제목` 은 사용하지 않습니다.
제목의 경우엔

```md
---
title: 제목
---

<!-- 문서에 대한 간략한 설명 -->

## 첫 섹션 제목
```

로 작성합니다.

`#` 은 빌드 파일에 자동생성되기 때문입니다.
raw 마크다운 파일로 렌더링된다할지라도 일관성을 위해 수동으로 작성 시 `#` 은 사용하지 않습니다.

주로 `##` 헤더를 사용하기 이전에 문서에 대한 설명을 간략하게 작성합니다

## 텍스트 스타일링 금지

볼드 처리를 할 때 헤더 제목 전체를 볼드 처리를 하지 않습니다.

```md
<!-- Bad: -->
## **볼드 제목**

<!-- Good: -->
## 그냥 제목
```

## 제목 작성 규칙

### ATX 스타일 사용

`===` 나 `---` 기호로 밑줄을 긋는 방식은 유지보수가 어려우므로 사용하지 않습니다.
반드시 `#` `##` `###` 기호를 사용합니다.

### 고유하고 완전한 제목

`### 요약` 대신 `### Foo 요약` 처럼 구체적으로 작성하여, 자동으로 생성되는 앵커 링크가 겹치지 않게 합니다.

### 충분한 여백

`#` 기호 뒤에는 반드시 띄어쓰기를 하고, 제목 위아래로 빈 줄을 하나씩 두어 코드 원문에서의 가독성을 높입니다.
