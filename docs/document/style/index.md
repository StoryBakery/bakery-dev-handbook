문서 작성 스타일에 대한 가이드입니다.

## 문법

문장 마지막에 점은 무조건 찍습니다.

### 들여쓰기

들여쓰기 시 공백 2칸을 사용합니다.
각 케이스에 대한 들여쓰기 규칙은 하위 문서들을 참고해주세요.

## 문서 구조와 목차

문서는 일관된 레이아웃을 가져야 독자가 필요한 정보를 빠르게 찾을 수 있습니다.

표준 레이아웃 순서

1. 제목
2. 짧은 소개
3. 본문 주제(H2)
4. 같이 보기(See also)

짧은 소개글 작성: 해당 주제를 전혀 모르는 초보자를 위해 1~3문장으로 개념과 목적을 요약합니다.

## 줄 길이와 텍스트 포맷팅

코드를 작성할 때의 좋은 습관을 문서 작성에도 동일하게 적용합니다.

### 한 줄에 80자 제한

한 줄은 최대 80자로 제한합니다. 단, 링크, 테이블, 제목, 코드 블록은 80자를 넘겨도 허용됩니다.

### 단락 구분

줄을 바꿀 때는 빈 줄(엔터 두 번)을 사용하여 새로운 단락(`<p>`)을 만드는 것이 가장 좋습니다.

## 테이블과 이미지

테이블 사용 최소화: 데이터가 균일하게 분포되어 있고 병렬적인 비교가 필요할 때만 테이블을 사용하세요. 내용이 길거나 빈칸이 많다면 '목록(List)' 형태로 푸는 것이 훨씬 읽기 좋습니다.

테이블 내 참조 링크: 테이블 셀의 길이가 80자를 훌쩍 넘지 않도록, 테이블 내부의 링크는 가급적 참조 링크로 처리합니다.

이미지 사용 최소화: 복잡한 UI를 설명할 때처럼 글보다 사진이 더 직관적일 때만 제한적으로 이미지를 사용하며, 반드시 시각장애인을 위한 대체 텍스트(Alt text)를 작성합니다.

## 순수 마크다운 우선

HTML 핵 지양: 마크다운 문법으로 해결할 수 없는 특이한 디자인을 위해 HTML 태그를 섞어 쓰는 것을 피하세요.
이는 문서의 이식성을 떨어뜨리고 다른 도구와의 연동을 방해합니다.
큰 테이블을 제외하면 대부분 순수 마크다운으로 해결 가능합니다.
